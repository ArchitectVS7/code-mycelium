# Contributing to Code Mycelium

Thanks for your interest in contributing! 🍄

## Development Status

**Current Phase:** Phase 1 Prototype (Complete)

We're exploring how to reveal hidden connections in codebases using mycelial network metaphors.

## How to Contribute

### Ideas & Feedback
- Open an issue describing what connections you'd like to reveal
- Share examples of code patterns you've discovered
- Suggest improvements to the visualization

### Code Contributions
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-connection`)
3. Make your changes
4. Test with `npm run demo`
5. Commit with descriptive messages
6. Push and open a Pull Request

### Areas for Contribution

**Phase 2 priorities:**
- LLM embeddings integration (OpenAI/Anthropic)

**Documentation:**
- More example codebases

**Visualization:**

## Code Style

- Use ES6+ features
- Keep functions focused (mycelial philosophy: small, connected)
- Comment non-obvious logic
- Run prettier before committing

## Philosophy

Code Mycelium is about **revealing**, not **generating**. We help developers see what's already there.

**Good contributions:**
- Show invisible connections
- Detect emergent patterns
- Surface hidden risks

**Not our focus:**
- Automated refactoring (we suggest, you decide)
- Code generation (there are other tools for that)
- Prescriptive architecture (we reveal, not enforce)

## Questions?

Open an issue or start a discussion. We're explorers in this space.
